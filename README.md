Este proyecto presenta el desarrollo de un planificador turístico que genera itinerarios <br> optimizados para recorrer la ciudad de Buenos Aires partiendo desde distintos hoteles <br> y seleccionando un conjunto de atracciones culturales y urbanas. La ciudad se modela <br> como un grafo donde los nodos representan puntos de interés, ya sean hoteles o <br> atracciones, y las aristas indican el tiempo estimado de traslado entre ellos. Para <br> calcular las rutas más eficientes se implementa el algoritmo de Dijkstra, considerando <br> no solo los traslados que se realizan en auto sino también el tiempo estimado de visita<br> de cada atracción. El sistema fue integrado en una aplicación web desarrollada con <br> Flask, que permite al usuario elegir el hotel de inicio, seleccionar las atracciones <br> deseadas y visualizar el itinerario detallado por día y enlaces directos a Google Maps.
